ISAM/ISVA Deployment & ISIM+AD Integration

Access Management

Deployed ISAM appliance with dual-network interface architecture for administrative (LMI) and runtime (Web Reverse Proxy) traffic isolation, integrated with ISIM and Active Directory.

View Full Deployment Blueprint (5 Phases + Integration)

🏠 Network Architecture Design:

ISAM appliance utilizes separate virtual network interfaces (NICs) to isolate administrative traffic from user application traffic. Logical design follows enterprise segmentation best practices:

Management Interface (eth0): Dedicated NIC for Local Management Interface (LMI) dashboard and SSH console access
Runtime Interface (eth1): Dedicated NIC for Web Reverse Proxy (WRP) junctions handling employee/user traffic
ISIM Integration Range: Pre-configured IP range for identity lifecycle management and provisioning source
User Registry (LDAP): Windows Server instance acting as primary user registry for ISAM authentication

🚀 Phase-by-Phase Deployment Blueprint:

  1. Phase 1: OVA Template Deployment (Hypervisor Setup)

    Deployed ISAM/ISVA OVA file in VMware Workstation/ESXi hypervisor. Provisioned 2 network adapters mapped to lab port group/vSwitch (bridged to internal lab network). Configured Adapter 1 for management traffic and Adapter 2 for application routing. Powered on VM.

  2. Phase 2: Initial Boot & Hardware CLI Setup

    Opened VM console, logged in with default credentials (admin/admin), acknowledged license agreement, and changed admin password. Entered CLI configuration wizard for network baseline setup.

  3. Phase 3: Management IP Baseline Assignment

    Configured management interface (management1/eth0) with static IP configuration: IP Type set to Static, assigned lab IP address, subnet mask, and gateway. Pressed Ctrl+C to save and apply settings. LMI engine restarted automatically.

  4. Phase 4: Web LMI Setup Wizard

    Navigated to LMI web interface via browser (HTTPS). Initial Setup Wizard launched automatically. Defined appliance hostname (e.g., isam-lab.local), configured primary and alternate DNS servers, activated core modules: Policy Server and Web Reverse Proxy.

  5. Phase 5: Application Interface Configuration

    Accessed system settings via LMI dashboard, enabled second interface (eth1), assigned static application IP for runtime traffic. Saved and deployed changes. Verified interface separation for LMI vs. WRP traffic.

🔗 Core Integration Configuration:

1. Directory Server Integration (External LDAP):

  • Targeted Windows Server instance as primary user registry
  • Configured LDAP connection: Host, Port (389 for standard, 636 for SSL), Administrator DN
  • Validated LDAP authentication pathway for ISAM runtime component

2. ISIM Connection Loop (Policy Server):

  • Navigated to Secure Web Settings > Manage > Policy Server
  • Ensured security sockets listening for incoming connection handshakes from ISIM subnet
  • Enabled ISIM-to-ISAM provisioning loop for automated access control management
  • Validated cross-component communication for identity lifecycle synchronization
