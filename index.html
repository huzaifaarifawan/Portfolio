IBM ISIM & Active Directory Full Deployment

IAM Case Study

End-to-end enterprise deployment of ISIM integrated with AD for automated identity lifecycle management (Joiner/Mover/Leaver).

View Full Deployment Details (8 Phases)

Environment Summary:

Platform: VMware Workstation
Network: 192.168.10.0/24 (Bridged)
ISIM: IBM Security Verify Governance
AD: Windows Server 2016+
Database: IBM DB2
LDAP: IBM SDS 6.4
SDI: 7.2 with RMI Dispatcher
Solution Dir: timsol

Architecture Overview:

ISIM Appliance → DB2 (Identity Backend)
             ↓
         LDAP (SDS 6.4) → Central User Repo
             ↓
          SDI 7.2 → Provisioning Engine
             ↓
    RMI Dispatcher → AD Adapter
             ↓
      Active Directory → Managed Resource

8-Phase Deployment Sequence:

  1. Phase 1: VMware Infrastructure Setup: Provisioned VMs, configured bridged networking (192.168.10.0/24), assigned static IPs, validated DNS configuration.
  2. Phase 2: IBM DB2 Deployment: Installed DB2 instance, configured ISIM schema, initialized database for identity backend storage.
  3. Phase 3: IBM SDS 6.4 (LDAP): Stood up LDAP directory server, configured solution directory (timsol), validated directory structure and replication.
  4. Phase 4: ISIM/ISVG Appliance: Installed ISIM virtual appliance, configured JVM properties, mail servers, workflow engines, hardened security settings.
  5. Phase 5: SDI 7.2 + RMI Dispatcher: Deployed SDI with RMI Dispatcher on Windows Server, configured Active Directory Adapter, mapped user/group attributes.
  6. Phase 6: Active Directory Setup: Deployed Windows Server AD, created domain controller, organizational units (OUs), user accounts, group policies.
  7. Phase 7: Integration & Provisioning: Integrated AD with ISIM as managed resource, configured automated User Provisioning/De-provisioning workflows.
  8. Phase 8: Policy Definition: Established Provisioning Policies, Identity Policies, Lifecycle Rules (Joiner/Mover/Leaver) for automated account creation based on AD groups.

Key Technical Decisions & Troubleshooting:

  • JVM isolation for ISIM appliance stability under load
  • RMI port behavior for Dispatcher-AD communication (port validation)
  • VM snapshot strategy for rapid recovery during testing
  • Internal DNS approach for network resolution
  • AD as central identity source for both ISIM provisioning and ISAM authentication

ISAM/ISVA Deployment & ISIM+AD Integration

Access Management

Deployed ISAM appliance with dual-network interface architecture for administrative (LMI) and runtime (Web Reverse Proxy) traffic isolation.

View Full Blueprint (5 Phases)

Network Architecture Design:

ISAM appliance utilizes separate virtual network interfaces (NICs) to isolate administrative traffic from user application traffic.

Management Interface (eth0): Dedicated NIC for LMI dashboard and SSH console
Runtime Interface (eth1): Dedicated NIC for Web Reverse Proxy junctions
ISIM Range: Pre-configured IP range for identity lifecycle management
User Registry (LDAP): Windows Server for ISAM authentication

5-Phase Deployment:

  1. Phase 1: OVA Template Deployment: Deployed ISAM/ISVA OVA in VMware Workstation/ESXi. Provisioned 2 network adapters mapped to lab vSwitch.
  2. Phase 2: Initial Boot & CLI Setup: Opened VM console, logged in (admin/admin), acknowledged license, changed admin password, entered CLI wizard.
  3. Phase 3: Management IP Assignment: Configured management interface (eth0) with static IP, subnet mask, gateway. Saved and restarted LMI.
  4. Phase 4: Web LMI Setup Wizard: Navigated to LMI (HTTPS), defined hostname, configured DNS, activated Policy Server and Web Reverse Proxy modules.
  5. Phase 5: Application Interface Configuration: Enabled second interface (eth1), assigned static application IP, verified interface separation.

Core Integration:

  • Directory Server (LDAP): Targeted Windows Server as primary user registry (Port 389/636)
  • ISIM Connection: Enabled security sockets for incoming handshakes from ISIM subnet
  • Provisioning Loop: Enabled ISIM-to-ISAM for automated access control management
