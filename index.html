📐 Architecture Overview:

ISIM Appliance → DB2 (Identity Backend)
             ↓
         LDAP (SDS 6.4) → Central User Repo
             ↓
          SDI 7.2 → Provisioning Engine
             ↓
    RMI Dispatcher → AD Adapter
             ↓
      Active Directory → Managed Resource

🚀 8-Phase Deployment Sequence:

  1. Phase 1: VMware Infrastructure Setup - Provisioned VMs, configured bridged networking (192.168.10.0/24), assigned static IPs, validated DNS
  2. Phase 2: IBM DB2 Deployment - Installed DB2 instance, configured ISIM schema, initialized database for identity backend
  3. Phase 3: IBM SDS 6.4 (LDAP) - Stood up LDAP directory server, configured solution directory (timsol), validated directory structure
  4. Phase 4: ISIM/ISVG Appliance - Installed ISIM virtual appliance, configured JVM properties, mail servers, workflow engines, hardened security
  5. Phase 5: SDI 7.2 + RMI Dispatcher - Deployed SDI with RMI Dispatcher on Windows Server, configured Active Directory Adapter, mapped attributes
  6. Phase 6: Active Directory Setup - Deployed Windows Server AD, created domain controller, OUs, user accounts, group policies
  7. Phase 7: Integration & Provisioning - Integrated AD with ISIM as managed resource, configured automated User Provisioning/De-provisioning
  8. Phase 8: Policy Definition - Established Provisioning Policies, Identity Policies, Lifecycle Rules (Joiner/Mover/Leaver) for automated account creation based on AD groups

✅ Verified Components: ISIM Appliance, DB2, LDAP, SDI, Dispatcher | 🔄 In Progress: ISAM integration with ISIM+AD

💡 Key Technical Decisions:

  • JVM isolation for ISIM appliance stability
  • RMI port behavior for Dispatcher-AD communication
  • Snapshot strategy for VM recovery
  • DNS approach for internal network resolution
  • AD as central identity source for both ISIM provisioning and ISAM authentication

IBM ISAM/ISVA Deployment & ISIM+AD Integration

Access Management

Deployed ISAM appliance with dual-network interfaces for LMI administration and Web Reverse Proxy runtime, integrated with ISIM and Active Directory for policy-based access enforcement.

View Full Deployment Blueprint (5 Phases + Integration)

🏠 Home Lab IP & Subnet Architecture:

Component / Interface IP Address Target Function
ISAM Management (eth0) 192.168.10.220 LMI dashboard & SSH console
ISAM Runtime/WRP (eth1) 192.168.10.221 Web Reverse Proxy junctions
ISIM Range 192.168.10.200-202 Identity lifecycle management
Windows Server (ISDS/SDI) 192.168.10.210 User registry (LDAP) for ISAM auth

Network: 255.255.255.0 | Gateway: 192.168.10.1 | DNS: 192.168.10.205

🚀 Phase-by-Phase Deployment Blueprint:

  1. Phase 1: Deploy OVA Template (Hypervisor Setup)

    Deploy ISAM/ISVA OVA file in VMware Workstation/ESXi. Provision 2 network adapters: Adapter 1 → Lab port group/vSwitch (bridged to 192.168.10.x), Adapter 2 → Same network for application routing. Power on VM.

  2. Phase 2: Initial Boot & Hardware CLI Setup (Console Access)

    Open VM console, log in (admin/admin), read license agreement, change admin password. Enter CLI configuration wizard.

  3. Phase 3: Assign Management IP Baseline (LMI Networking)

    Configure management interface (management1/eth0):

    • IP Type: Static
    • IP Address: 192.168.10.220
    • Subnet Mask: 255.255.255.0
    • Gateway: 192.168.10.1

    Press Ctrl+C to save → LMI engine restarts.

  4. Phase 4: Run Setup Wizard via Web LMI (Browser Wizard)

    Navigate to https://192.168.10.220:443, log in with new credentials. Initial Setup Wizard starts automatically:

    • Define hostname: isam-lab.local
    • DNS: 192.168.10.205 | Alternate DNS: 8.
